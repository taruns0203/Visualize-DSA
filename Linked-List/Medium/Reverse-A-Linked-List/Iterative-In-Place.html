üìù The Algorithm (4 lines of magic!)

curr = head

next = curr.next // 1. Save next
curr.next = prev // 2. Reverse pointer

prev = Previous node (already reversed)
curr = Current node (processing now)
next = Next node (saved before we break link)
Key Insight
We must save curr.next BEFORE we change it! Otherwise we lose access to the rest of the list. Think of it like a train: we're uncoupling cars and reattaching them in reverse order.
